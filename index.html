fr fr, networking used to be wild back in the day 💀 Before we had this cool CIDR stuff, we were stuck with something called class-based networking and ngl, it was kinda mid

so here's the tea ☕️ class-based networking split IPs into these fixed sizes:

  • Class A: first bit is 0 (0.0.0.0 to 127.255.255.255) - way too huge tbh
  • Class B: first bits are 10 (128.0.0.0 to 191.255.255.255) - still massive fr
  • Class C: first bits are 110 (192.0.0.0 to 223.255.255.255) - sometimes too small no cap

ngl the whole class system was lowkey a mess 😩

  • 💀 imagine needing 1000 IPs but getting stuck with either 256 (too small) or 65,536 (way too big)
  • 🤡 wasting mad addresses cause you couldn't get the size you actually needed
  • 📉 internet routing tables getting bloated af

but then CIDR came thru and changed the game 😤✨ Instead of being stuck with these preset sizes, you can just say exactly how many bits you want for the network part!

like when you see 192.168.1.0/24, that just means "use 24 bits for the network" - super clean and flexible no cap 💯

try it out below and see how much better this is fr fr 🔥
